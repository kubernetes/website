Có thể chỉ đến: core Kubernetes hoặc source repo mà từ đó một repo được fork.

<!--more-->

* Trong **Kubernetes Community**: Các cuộc trò chuyện thường sử dụng *upstream* để chỉ core Kubernetes codebase, mà hệ sinh thái chung, code khác, hoặc các công cụ của bên thứ ba phụ thuộc vào. Ví dụ, [community members](#term-member) có thể đề xuất rằng một tính năng được chuyển upstream để nó nằm trong core codebase thay vì trong một plugin hoặc công cụ của bên thứ ba.
* Trong **GitHub** hoặc **git**: Quy ước là gọi source repo là *upstream*, trong khi repo được fork được coi là *downstream*.
