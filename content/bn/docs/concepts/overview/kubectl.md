{{< glossary_definition prepend="কুবারনেটিস একটি" term_id="kubectl" length="short" >}}

`kubectl` টুল [কুবারনেটিস API](/bn/docs/concepts/overview/kubernetes-api/) এর মাধ্যমে আপনার ক্লাস্টারের সাথে যোগাযোগ করে।
কনফিগারেশনের জন্য, `kubectl` `$HOME/.kube` ডিরেক্টরিতে `config` নামের একটি ফাইল খুঁজে।
আপনি `KUBECONFIG` এনভায়রনমেন্ট ভেরিয়েবল সেট করে বা
[`--kubeconfig`](/bn/docs/concepts/configuration/organize-cluster-access-kubeconfig/) ফ্ল্যাগ সেট করে অন্যান্য [kubeconfig](/bn/docs/concepts/configuration/organize-cluster-access-kubeconfig/)
ফাইল নির্দিষ্ট করতে পারেন।

<!-- body -->

## kubectl এর ভূমিকা

`kubectl` টুল হল কুবারনেটিস অবজেক্ট তৈরি, পরিদর্শন, আপডেট এবং মুছে ফেলার জন্য প্রাথমিক ইন্টারফেস।
এটি আপনার ক্লাস্টারের ভিতরে চলা [কুবারনেটিস কম্পোনেন্ট](/bn/docs/concepts/overview/components/) এবং
সেই কম্পোনেন্টগুলি যে [কুবারনেটিস API](/bn/docs/concepts/overview/kubernetes-api/) ইমপ্লিমেন্ট করে তার পরিপূরক।
আপনি আপনার ল্যাপটপ থেকে বা ক্লাস্টারের ভিতরে একটি Pod থেকে `kubectl` চালান না কেন, এটি API সার্ভারে অনুরোধ পাঠায়।
অন্যান্য ক্লায়েন্ট, যেমন [ক্লায়েন্ট লাইব্রেরি](/bn/docs/reference/using-api/client-libraries/) এবং ওয়েব ড্যাশবোর্ড
যেমন [Headlamp](https://headlamp.dev/), একই API এর মাধ্যমে যোগাযোগ করে।

## kubectl কীভাবে কাজ করে

`kubectl` টুল API সার্ভারের সাথে সংযুক্ত হয় এবং আপনার
[kubeconfig](/bn/docs/concepts/configuration/organize-cluster-access-kubeconfig/) ফাইলে সংজ্ঞায়িত ক্লাস্টার, ব্যবহারকারী এবং কনটেক্সট ব্যবহার করে অথেনটিকেট করে।
যখন আপনি একটি ক্লাস্টারের বাইরে থেকে `kubectl` চালান, এটি API সার্ভার ঠিকানা এবং ক্রেডেনশিয়াল খুঁজতে kubeconfig ফাইল ব্যবহার করে।
যখন `kubectl` একটি Pod এর ভিতরে চলে (উদাহরণস্বরূপ, একটি CI/CD পাইপলাইনে), এটি
Pod এ মাউন্ট করা ServiceAccount টোকেনের উপর ভিত্তি করে ইন-ক্লাস্টার অথেনটিকেশন ব্যবহার করতে পারে।

যখন আপনি একটি কমান্ড চালান, `kubectl` আপনার উদ্দেশ্যকে
[কুবারনেটিস API](/bn/docs/concepts/overview/kubernetes-api/) এ এক বা একাধিক HTTP অনুরোধে অনুবাদ করে। API সার্ভার প্রতিটি অনুরোধ ভ্যালিডেট করে,
এটি {{< glossary_tooltip text="etcd" term_id="etcd" >}} এ সংরক্ষিত ক্লাস্টার স্টেটে প্রয়োগ করে, এবং
ফলাফল রিটার্ন করে। এর মানে প্রতিটি `kubectl` অ্যাকশন, একটি Deployment তৈরি করা বা লগ পড়া হোক,
একই API-চালিত পাথ অনুসরণ করে।

যেহেতু আপনার kubeconfig একাধিক ক্লাস্টার, ব্যবহারকারী এবং কনটেক্সট সংজ্ঞায়িত করতে পারে, আপনি আপনার এনভায়রনমেন্ট পুনরায় কনফিগার না করে ক্লাস্টারের মধ্যে স্যুইচ করতে `kubectl` ব্যবহার করতে পারেন। সক্রিয় কনটেক্সট পরিবর্তন করতে `kubectl config use-context` চালান।

## kubectl দিয়ে আপনি কী করতে পারেন

`kubectl` টুল অনেক অপারেশন সমর্থন করে, যা এই বিস্তৃত বিভাগে পড়ে:

* **রিসোর্স পরিচালনা** – Pod, Deployment এবং Service এর মতো অবজেক্ট তৈরি, আপডেট এবং মুছুন।
  কনফিগারেশন ফাইল থেকে ডিক্লারেটিভ ম্যানেজমেন্টের জন্য `kubectl apply` ব্যবহার করুন।
* **ক্লাস্টার স্টেট পরিদর্শন** – অবজেক্ট তালিকা এবং বর্ণনা করুন, ইভেন্ট দেখুন এবং রিসোর্স ব্যবহার চেক করুন।
* **ডিবাগ** – কন্টেইনার থেকে লগ দেখুন, একটি চলমান কন্টেইনারের ভিতরে কমান্ড এক্সিকিউট করুন, বা একটি Pod এ পোর্ট-ফরওয়ার্ড করুন।
* **ক্লাস্টার অপারেশন** – রক্ষণাবেক্ষণের জন্য Node ড্রেইন করুন, নতুন ওয়ার্কলোড প্রতিরোধ করতে Node কর্ডন করুন এবং ক্লাস্টার কনফিগারেশন পরিচালনা করুন।
* **স্ক্রিপ্ট এবং অটোমেট** – স্ক্রিপ্ট এবং পাইপলাইনে ব্যবহারের জন্য [JSONPath](/bn/docs/reference/kubectl/jsonpath/) ব্যবহার করে JSON, YAML বা কাস্টম কলাম হিসাবে আউটপুট ফর্ম্যাট করুন।

সিনট্যাক্স, কমান্ড রেফারেন্স এবং উদাহরণের জন্য, [kubectl রেফারেন্স ডকুমেন্টেশন](/docs/reference/kubectl/) দেখুন।

## ডিক্লারেটিভ বনাম ইম্পারেটিভ

প্রোডাকশন ওয়ার্কলোডের জন্য, ভার্সন-কন্ট্রোলড কনফিগারেশন ফাইল সহ `kubectl apply` ব্যবহার করে [ডিক্লারেটিভ অবজেক্ট ম্যানেজমেন্ট](/bn/docs/concepts/overview/working-with-objects/object-management/)
পছন্দ করুন।
ডিক্লারেটিভ ম্যানেজমেন্ট আপনাকে পরিবর্তন ট্র্যাক করতে, সহযোগিতা করতে এবং GitOps ওয়ার্কফ্লোর সাথে ইন্টিগ্রেট করতে সহায়তা করে।
ইম্পারেটিভ কমান্ড (যেমন `kubectl create` বা `kubectl run`) ডেভেলপমেন্ট এবং এক্সপেরিমেন্টেশনের জন্য উপযোগী,
কিন্তু পুনরুৎপাদন এবং অডিট করা কঠিন।

## প্লাগইন দিয়ে kubectl প্রসারিত করা

আপনি [প্লাগইন](/bn/docs/tasks/extend-kubectl/kubectl-plugins/) দিয়ে `kubectl` প্রসারিত করতে পারেন যা নতুন
সাব-কমান্ড যোগ করে। প্লাগইন হল স্বতন্ত্র বাইনারি যা `kubectl-<plugin-name>` নামকরণ কনভেনশন অনুসরণ করে।
কুবারনেটিস কমিউনিটি অনেক প্লাগইন রক্ষণাবেক্ষণ করে, এবং আপনি সেগুলি
[Krew](https://krew.sigs.k8s.io/) প্লাগইন ম্যানেজার দিয়ে পরিচালনা করতে পারেন।

## ভার্সন সামঞ্জস্য

`kubectl` টুল ক্লাস্টারের কন্ট্রোল প্লেনের তুলনায় প্লাস-অর-মাইনাস এক মাইনর ভার্সনের একটি ভার্সন স্কিউ সমর্থন করে।
উদাহরণস্বরূপ, `kubectl` v1.32 v1.31, v1.32 এবং v1.33 এ কন্ট্রোল প্লেনের সাথে কাজ করে।
একটি সামঞ্জস্যপূর্ণ ভার্সন ব্যবহার করা অপ্রত্যাশিত আচরণ এড়ায়। বিস্তারিত জানতে
[ভার্সন স্কিউ পলিসি](/releases/version-skew-policy/) দেখুন।
