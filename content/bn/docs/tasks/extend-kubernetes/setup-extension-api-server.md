## অ্যাগ্রিগেশন লেয়ারের সাথে কাজ করার জন্য একটি এক্সটেনশন api-server সেট আপ করুন

নিম্নলিখিত ধাপগুলি বর্ণনা করে কীভাবে একটি এক্সটেনশন-apiserver *উচ্চ স্তরে* সেট আপ করতে হয়। আপনি YAML কনফিগ ব্যবহার করছেন বা API ব্যবহার করছেন তা নির্বিশেষে এই ধাপগুলি প্রযোজ্য। দুটির মধ্যে যেকোনো পার্থক্য বিশেষভাবে চিহ্নিত করার চেষ্টা করা হয়েছে। YAML কনফিগ ব্যবহার করে কীভাবে সেগুলি ইমপ্লিমেন্ট করা যেতে পারে তার একটি সুনির্দিষ্ট উদাহরণের জন্য, আপনি কুবারনেটিস রিপোতে [sample-apiserver](https://github.com/kubernetes/sample-apiserver/blob/master/README.md) দেখতে পারেন।

বিকল্পভাবে, আপনি একটি বিদ্যমান থার্ড পার্টি সমাধান ব্যবহার করতে পারেন, যেমন [apiserver-builder](https://github.com/kubernetes-sigs/apiserver-builder-alpha/blob/master/README.md), যা একটি স্কেলেটন জেনারেট করবে এবং আপনার জন্য নিম্নলিখিত সমস্ত ধাপ অটোমেট করবে।

1. নিশ্চিত করুন যে APIService API সক্ষম আছে (`--runtime-config` চেক করুন)। এটি ডিফল্টরূপে চালু থাকা উচিত, যদি না এটি আপনার ক্লাস্টারে ইচ্ছাকৃতভাবে বন্ধ করা হয়েছে।
1. আপনার APIService অবজেক্ট যোগ করার অনুমতি দেওয়ার জন্য একটি RBAC রুল তৈরি করতে হতে পারে, বা আপনার ক্লাস্টার অ্যাডমিনিস্ট্রেটরকে একটি তৈরি করতে বলুন। (যেহেতু API এক্সটেনশন সম্পূর্ণ ক্লাস্টারকে প্রভাবিত করে, একটি লাইভ ক্লাস্টারে API এক্সটেনশনের টেস্টিং/ডেভেলপমেন্ট/ডিবাগ করার সুপারিশ করা হয় না।)
1. আপনি যে কুবারনেটিস নেমস্পেসে আপনার এক্সটেনশন api-service চালাতে চান তা তৈরি করুন।
1. এক্সটেনশন api-server HTTPS এর জন্য যে সার্ভার সার্ট ব্যবহার করে তা সাইন করতে ব্যবহার করার জন্য একটি CA সার্ট তৈরি/পান।
1. api-server HTTPS এর জন্য ব্যবহার করার জন্য একটি সার্ভার সার্ট/কী তৈরি করুন। এই সার্টটি উপরের CA দ্বারা সাইন করা উচিত। এটিতে Kube DNS নামের একটি CN থাকা উচিত। এটি কুবারনেটিস সার্ভিস থেকে উদ্ভূত এবং `<service name>.<service name namespace>.svc` ফর্মের হওয়া উচিত
1. আপনার নেমস্পেসে সার্ভার সার্ট/কী সহ একটি কুবারনেটিস সিক্রেট তৈরি করুন।
1. এক্সটেনশন api-server এর জন্য একটি কুবারনেটিস ডিপ্লয়মেন্ট তৈরি করুন এবং নিশ্চিত করুন যে আপনি সিক্রেটটি একটি ভলিউম হিসাবে লোড করছেন। এতে আপনার এক্সটেনশন api-server এর একটি কার্যকর ইমেজের রেফারেন্স থাকা উচিত। ডিপ্লয়মেন্টটিও আপনার নেমস্পেসে থাকা উচিত।
1. নিশ্চিত করুন যে আপনার এক্সটেনশন-apiserver সেই ভলিউম থেকে সেই সার্টগুলি লোড করে এবং সেগুলি HTTPS হ্যান্ডশেকে ব্যবহৃত হয়।
1. আপনার নেমস্পেসে একটি কুবারনেটিস সার্ভিস অ্যাকাউন্ট তৈরি করুন।
1. আপনার রিসোর্সে আপনি যে অপারেশনগুলি অনুমতি দিতে চান তার জন্য একটি কুবারনেটিস ক্লাস্টার রোল তৈরি করুন।
1. আপনার নেমস্পেসের সার্ভিস অ্যাকাউন্ট থেকে আপনার তৈরি ক্লাস্টার রোলে একটি কুবারনেটিস ক্লাস্টার রোল বাইন্ডিং তৈরি করুন।
1. কুবারনেটিস কোর API সার্ভারে auth সিদ্ধান্ত ডেলিগেট করতে আপনার নেমস্পেসের সার্ভিস অ্যাকাউন্ট থেকে `system:auth-delegator` ক্লাস্টার রোলে একটি কুবারনেটিস ক্লাস্টার রোল বাইন্ডিং তৈরি করুন।
1. আপনার নেমস্পেসের সার্ভিস অ্যাকাউন্ট থেকে `extension-apiserver-authentication-reader` রোলে একটি কুবারনেটিস রোল বাইন্ডিং তৈরি করুন। এটি আপনার এক্সটেনশন api-server কে `extension-apiserver-authentication` কনফিগম্যাপ অ্যাক্সেস করার অনুমতি দেয়।
1. একটি কুবারনেটিস apiservice তৈরি করুন। উপরের CA সার্টটি base64 এনকোড করা উচিত, নতুন লাইন ছাড়ানো এবং apiservice এ spec.caBundle হিসাবে ব্যবহার করা উচিত। এটি নেমস্পেসড হওয়া উচিত নয়। যদি [kube-aggregator API](https://github.com/kubernetes/kube-aggregator/) ব্যবহার করেন, শুধুমাত্র PEM এনকোডেড CA বান্ডেল পাস করুন কারণ base 64 এনকোডিং আপনার জন্য করা হয়।
1. আপনার রিসোর্স পেতে kubectl ব্যবহার করুন। চালানো হলে, kubectl "No resources found." রিটার্ন করা উচিত। এই বার্তাটি
নির্দেশ করে যে সবকিছু কাজ করেছে কিন্তু আপনার বর্তমানে সেই রিসোর্স টাইপের কোনো অবজেক্ট তৈরি নেই।
