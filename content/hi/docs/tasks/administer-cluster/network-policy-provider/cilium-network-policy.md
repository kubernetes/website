## बेसिक टेस्टिंग के लिए मिनीक्यूब पर सिलियम डिप्लॉय करना

सिलियम से आसानी से परिचित होने के लिए आप
[सिलियम कुबेरनेट्स आरंभ करने की गाइड](https://docs.cilium.io/en/stable/gettingstarted/k8s-install-default/)
का पालन करके मिनीक्यूब में सिलियम का बेसिक डेमनसेट इंस्टॉलेशन कर सकते हैं।

मिनीक्यूब शुरू करने के लिए, जिसके लिए v1.5.2 या उससे ऊपर का वर्शन आवश्यक है, निम्नलिखित आर्गुमेंट्स के साथ चलाएं:

```shell
minikube version
```

```
minikube version: v1.5.2
```

```shell
minikube start --network-plugin=cni
```

मिनीक्यूब के लिए आप सिलियम को उसके सीएलआई टूल का उपयोग करके इंस्टॉल कर सकते हैं। ऐसा करने के लिए, पहले निम्नलिखित कमांड से सीएलआई का नवीनतम वर्शन डाउनलोड करें:

```shell
curl -LO https://github.com/cilium/cilium-cli/releases/latest/download/cilium-linux-amd64.tar.gz
```

फिर डाउनलोड की गई फ़ाइल को निम्नलिखित कमांड से अपनी `/usr/local/bin` डायरेक्टरी में एक्सट्रैक्ट करें:

```shell
sudo tar xzvfC cilium-linux-amd64.tar.gz /usr/local/bin
rm cilium-linux-amd64.tar.gz
```

ऊपर दिए गए कमांड चलाने के बाद, अब आप निम्नलिखित कमांड से सिलियम इंस्टॉल कर सकते हैं:

```shell
cilium install
```

सिलियम स्वचालित रूप से क्लस्टर कॉन्फ़िगरेशन का पता लगाएगा और सफल इंस्टॉलेशन के लिए उचित कंपोनेंट्स बनाएगा और इंस्टॉल करेगा।
कंपोनेंट्स हैं:

- सीक्रेट `cilium-ca` में सर्टिफिकेट अथॉरिटी (सीए) और हबल (सिलियम की ऑब्ज़र्वेबिलिटी लेयर) के लिए सर्टिफिकेट।
- सर्विस अकाउंट्स।
- क्लस्टर रोल्स।
- कॉन्फ़िगमैप।
- एजेंट डेमनसेट और एक ऑपरेटर डिप्लॉयमेंट।

इंस्टॉलेशन के बाद, आप `cilium status` कमांड से सिलियम डिप्लॉयमेंट की समग्र स्थिति देख सकते हैं।
`status` कमांड का अपेक्षित आउटपुट
[यहाँ](https://docs.cilium.io/en/stable/gettingstarted/k8s-install-default/#validate-the-installation) देखें।

शेष आरंभ करने की गाइड एक उदाहरण एप्लिकेशन का उपयोग करके एल3/एल4
(यानी, आईपी एड्रेस + पोर्ट) सिक्योरिटी पॉलिसीज़ और एल7 (जैसे, एचटीटीपी) सिक्योरिटी
पॉलिसीज़ दोनों को लागू करने का तरीका बताती है।

## प्रोडक्शन उपयोग के लिए सिलियम डिप्लॉय करना

प्रोडक्शन के लिए सिलियम डिप्लॉय करने के विस्तृत निर्देशों के लिए, देखें:
[सिलियम कुबेरनेट्स इंस्टॉलेशन गाइड](https://docs.cilium.io/en/stable/network/kubernetes/concepts/)
इस डॉक्यूमेंटेशन में विस्तृत आवश्यकताएं, निर्देश और उदाहरण
प्रोडक्शन डेमनसेट फ़ाइलें शामिल हैं।

## सिलियम कंपोनेंट्स को समझना

सिलियम के साथ क्लस्टर डिप्लॉय करने से `kube-system` नेमस्पेस में पॉड्स जुड़ जाते हैं। इन
पॉड्स की सूची देखने के लिए चलाएं:

```shell
kubectl get pods --namespace=kube-system -l k8s-app=cilium
```

आपको इसके जैसी पॉड्स की सूची दिखेगी:

```console
NAME           READY   STATUS    RESTARTS   AGE
cilium-kkdhz   1/1     Running   0          3m23s
...
```

एक `cilium` पॉड आपके क्लस्टर के प्रत्येक नोड पर चलता है और लिनक्स बीपीएफ का उपयोग करके
उस नोड पर पॉड्स के ट्रैफ़िक पर नेटवर्क पॉलिसी लागू करता है।
