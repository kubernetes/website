नेटवर्किंग Kubernetes का एक महत्वपूर्ण हिस्सा है, लेकिन यह समझना थोड़ा कठिन हो सकता है कि यह वास्तव में कैसे काम करता है। इसमें 4 मुख्य नेटवर्किंग समस्याएँ होती हैं:

- कंटेनर-से-कंटेनर संचार: इसे Pods और localhost संचार द्वारा हल किया जाता है।
- Pod-से-Pod संचार: यह इस दस्तावेज़ का मुख्य विषय है।
- Pod-से-Service संचार: इसे Services द्वारा संभाला जाता है।
- बाहरी-से-Service संचार: इसे भी Services द्वारा संभाला जाता है।

Kubernetes का मुख्य उद्देश्य मशीनों को विभिन्न एप्लिकेशन के बीच साझा करना है। आमतौर पर, मशीन साझा करने के लिए यह सुनिश्चित करना आवश्यक होता है कि दो एप्लिकेशन एक ही पोर्ट का उपयोग न करें। कई डेवलपर्स के बीच पोर्ट का समन्वय करना बहुत कठिन होता है और यह उपयोगकर्ताओं को क्लस्टर-स्तरीय समस्याओं के संपर्क में ला सकता है।

डायनामिक पोर्ट एलोकेशन सिस्टम को और जटिल बना देता है - हर एप्लिकेशन को पोर्ट्स को फ्लैग्स के रूप में लेना पड़ता है, API सर्वर को कॉन्फ़िगरेशन में पोर्ट डालना होता है, और Services को एक-दूसरे को ढूंढना होता है। इन जटिलताओं से बचने के लिए, Kubernetes एक अलग तरीका अपनाता है।

Kubernetes नेटवर्किंग मॉडल के बारे में जानने के लिए, संबंधित दस्तावेज़ देखें।

## Kubernetes IP address ranges

Kubernetes क्लस्टर को Pods, Services और Nodes के लिए अलग-अलग IP एड्रेस रेंज की आवश्यकता होती है। ये निम्नलिखित कंपोनेंट्स द्वारा कॉन्फ़िगर किए जाते हैं:

- नेटवर्क प्लगइन Pods को IP एड्रेस असाइन करता है।
- kube-apiserver Services को IP एड्रेस असाइन करता है।
- kubelet या cloud-controller-manager Nodes को IP एड्रेस असाइन करता है।

## Cluster networking types

Kubernetes क्लस्टर, कॉन्फ़िगर किए गए IP परिवार के आधार पर निम्न प्रकार के हो सकते हैं:

- **IPv4 only**: केवल IPv4 एड्रेस उपयोग किए जाते हैं।
- **IPv6 only**: केवल IPv6 एड्रेस उपयोग किए जाते हैं।
- **Dual-stack (IPv4/IPv6)**:
  - नेटवर्क प्लगइन IPv4 और IPv6 दोनों असाइन करता है।
  - kube-apiserver दोनों प्रकार के एड्रेस असाइन करता है।
  - kubelet या cloud-controller-manager दोनों एड्रेस असाइन करता है।

सभी कंपोनेंट्स को प्राथमिक IP परिवार पर सहमत होना आवश्यक है।

Kubernetes केवल Pods, Services और Nodes में मौजूद IP एड्रेस को ही ध्यान में रखता है, न कि वास्तविक सिस्टम के सभी IPs को।

## How to implement the Kubernetes network model

नेटवर्क मॉडल प्रत्येक नोड पर कंटेनर रनटाइम द्वारा लागू किया जाता है। अधिकतर कंटेनर रनटाइम नेटवर्क और सुरक्षा को मैनेज करने के लिए CNI (Container Network Interface) प्लगइन्स का उपयोग करते हैं।

कई प्रकार के CNI प्लगइन्स उपलब्ध हैं:
- कुछ केवल बेसिक नेटवर्किंग प्रदान करते हैं
- कुछ उन्नत सुविधाएँ देते हैं जैसे:
  - मल्टी-CNI सपोर्ट
  - IP मैनेजमेंट
  - अन्य सिस्टम्स के साथ इंटीग्रेशन

अधिक जानकारी के लिए Kubernetes नेटवर्किंग एडऑन सूची देखें।

## आगे क्या है

अधिक जानकारी के लिए Kubernetes नेटवर्किंग डिज़ाइन डॉक्यूमेंट और SIG-Network KEPs देखें।
