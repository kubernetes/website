Wenn du ein Problem in der Kubernetes-Dokumentation feststellst oder eine Idee für neue Inhalte hast, öffne ein Issue. Alles, was du brauchst, ist ein [GitHub-Konto](https://github.com/join) und einen Webbrowser.

In den meisten Fällen beginnt neue Arbeit an der Kubernetes-Dokumentation mit einem Issue auf GitHub. Kubernetes-Mitwirkende prüfen, kategorisieren und kennzeichnen Issues nach Bedarf. Anschließend öffnest du oder ein anderes Mitglied der Kubernetes-Community einen Pull Request mit Änderungen, um das Issue zu lösen.

<!-- body -->

## Ein Issue öffnen

Wenn du Verbesserungen an bestehenden Inhalten vorschlagen oder einen Fehler melden möchtest, öffne ein Issue.

1. Klicke auf den Link **Create an issue** in der rechten Seitenleiste. Du wirst zu einer GitHub-Issue-Seite weitergeleitet, die mit einigen Headern vorausgefüllt ist.
2. Beschreibe das Problem oder den Verbesserungsvorschlag. Gib so viele Details wie möglich an.
3. Klicke auf **Submit new issue**.

Überprüfe nach dem Einreichen gelegentlich dein Issue oder aktiviere GitHub-Benachrichtigungen.
Reviewer und andere Community-Mitglieder könnten Fragen stellen, bevor sie Maßnahmen ergreifen können.

## Neue Inhalte vorschlagen

Wenn du eine Idee für neue Inhalte hast, aber nicht sicher bist, wo sie hingehören, kannst du trotzdem ein Issue einreichen. Entweder:

- Wähle eine bestehende Seite in dem Abschnitt, in dem du denkst, dass der Inhalt hingehört, und klicke auf **Create an issue**.
- Gehe direkt zu [GitHub](https://github.com/kubernetes/website/issues/new/) und reiche das Issue dort ein.

## Wie man gute Issues erstellt

Beachte beim Einreichen eines Issues Folgendes:

- Gib eine klare Beschreibung des Issues. Beschreibe, was genau fehlt, veraltet, falsch oder verbesserungswürdig ist.
- Erkläre die spezifischen Auswirkungen des Issues auf Nutzer.
- Begrenze den Umfang eines Issues auf eine sinnvolle Arbeitseinheit. Teile Probleme mit großem Umfang in kleinere Issues auf. Zum Beispiel ist „Sicherheitsdokumentation verbessern" zu weit gefasst, aber „Details zum Thema ‚Netzwerkzugriff einschränken' hinzufügen" ist konkret genug.
- Durchsuche die bestehenden Issues, um zu sehen, ob es etwas Ähnliches oder Verwandtes gibt.
- Wenn das neue Issue mit einem anderen Issue oder Pull Request zusammenhängt, verweise darauf entweder über die vollständige URL oder die Issue- bzw. Pull-Request-Nummer mit einem vorangestellten `#`-Zeichen. Zum Beispiel: `Eingeführt durch #987654`.
- Halte dich an den [Verhaltenskodex](/community/code-of-conduct/). Respektiere deine Mitarbeitenden. Zum Beispiel ist „Die Dokumentation ist schrecklich" kein hilfreicher oder höflicher Kommentar.
