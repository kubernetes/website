挑戰

其遊戲業務是世界上最大的遊戲業務之一，但這不是 NetEase 爲中國消費者提供的所有。公司還經營電子商務、廣告、音樂流媒體、在線教育和電子郵件平臺；其中最後一個服務有近10億使用者通過網站使用免費的電子郵件服務，如 163.com。在2015 年，爲所有這些系統提供基礎設施的 NetEase Cloud 團隊意識到，他們的研發流程正在減緩開發人員的速度。NetEase Cloud 和容器服務架構師 Feng Changjian 表示：“我們的使用者需要自己準備所有基礎設施。”“我們希望通過無伺服器容器服務自動爲使用者提供基礎設施和工具。”

解決方案

在考慮構建自己的業務流程解決方案後，NetEase 決定將其私有云平臺建立在 Kubernetes 的基礎上。這項技術來自 Google，這一事實讓團隊有信心，它能夠跟上 NetEase 的規模。“經過2到3個月的評估，我們相信它能滿足我們的需求，”Feng Changjian 說。該團隊於 2015 年開始與 Kubernetes 合作，那會它甚至還不是 1.0 版本。如今，NetEase 內部雲平臺還使用了 CNCF 項目 Prometheus、Envoy、Harbor、gRPC 和 Helm， 在生產叢集中運行 10000 個節點，並可支持叢集多達 30000 個節點。基於對內部平臺的學習，公司向外部客戶推出了基於 Kubernetes 的雲和微服務型 PaaS 產品，NetEase 輕舟微服務。

影響

NetEase 團隊報告說，Kubernetes 已經提高了研發效率一倍多，部署效率提高了 2.8 倍。“過去，如果我們想要進行升級，我們需要與其他團隊合作，甚至加入其他部門，”Feng Changjian 說。“我們需要專人來準備一切，需要花費約半個小時。現在我們只需 5 分鐘即可完成。”新平臺還允許使用 GPU 和 CPU 資源進行混合部署。“以前，如果我們將所有資源都用於 GPU，則 CPU 的備用資源將沒有。但是現在，由於混合部署，我們有了很大的改進，”他說。這些改進也提高了資源的利用率。

該團隊於 2015 年開始採用 Kubernetes，那會它甚至還不是 1.0 版本，因爲它相對易於使用，並且使 DevOps 在公司中得以實現。“我們放棄了 Kubernetes 的一些概念；我們只想使用標準化框架，”Feng Changjian 說。“我們利用 Kubernetes 的可編程性，構建一個平臺，以滿足內部客戶對升級和部署的需求。”

團隊首先專注於構建容器平臺以更好地管理資源，然後通過添加內部系統（如監視）來改進對微服務的支持。這意味着整合了 CNCF 項目 Prometheus，Envoy，Harbor，gRPC 和 Helm。“我們正在努力提供簡化和標準化的流程，以便我們的使用者和客戶能夠利用我們的最佳實踐，”Feng Changjian 說。

團隊正在繼續改進。例如，企業的電子商務部分需要利用混合部署，過去需要使用兩個單獨的平臺：基礎架構即服務平臺和 Kubernetes 平臺。最近，NetEase 創建了一個跨平臺應用程式，支持將兩者同時使用單命令部署。

{{< case-studies/quote image="/images/case-studies/netease/banner4.jpg" author="Li Lanqing，NetEase Kubernetes 開發人員" >}} “只要公司擁有成熟的團隊和足夠的開發人員，我認爲 Kubernetes 是一個很好的有所助力的技術。” {{< /case-studies/quote >}}

{{< case-studies/quote author="Li Lanqing，NetEase Kubernetes 開發人員">}} “通過與這個社區接觸，我們可以從中獲得一些經驗，我們也可以從中獲益。我們可以看到社區所關心的問題和挑戰，以便我們參與其中。” {{< /case-studies/quote >}}

基於使用內部平臺的成果和學習，公司向外部客戶推出了基於 Kubernetes 的雲和微服務型 PaaS 產品，NetEase 輕舟微服務。“我們的想法是，我們可以找到我們的遊戲和電子商務以及雲音樂提供商遇到的問題，所以我們可以整合他們的體驗，並提供一個平臺，以滿足所有使用者的需求，”Zeng Yuxing 說。

無論是否使用 NetEase 產品，該團隊鼓勵其他公司嘗試 Kubernetes。Kubernetes 開發者 Li Lanqing 表示：“只要公司擁有成熟的團隊和足夠的開發人員，我認爲 Kubernetes 是一個很好的技術，可以幫助他們。”

作爲最終使用者和供應商，NetEase 已經更多地參與社區，向其他公司學習，分享他們所做的工作。該團隊一直在爲 Harbor 和 Envoy 項目做出貢獻，在 NetEase 進行規模測試技術時提供反饋。“我們是一個團隊，專注於應對微服務架構的挑戰，”Feng Changjian 說。“通過與這個社區接觸，我們可以從中獲得一些經驗，我們也可以從中獲益。我們可以看到社區所關心的問題和挑戰，以便我們參與其中。”
