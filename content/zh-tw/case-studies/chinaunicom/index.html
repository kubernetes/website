挑戰

中國聯通是中國三大電信運營商之一，爲了服務其 3 億用戶，該公司運營着多個數據中心，每個數據中心有數千臺服務器， 自 2016 年以來使用 Docker 容器化以及 VMWare 和 OpenStack 基礎設施。 不巧的是，“資源利用率相對較低”，平臺技術研發組組長張承宇表示，“以前沒有一個雲平臺可以承載我們的數百個應用。” 中國聯通曾是一家完全國有的公司，近年來接受了來自 BAT（百度、阿里巴巴、騰訊）和京東的民營投資， 現在專注於使用開源技術而不是商業產品進行內部開發。因此，張承宇的中國聯通實驗室團隊開始尋找適合其雲基礎設施的開源編排方案。

解決方案

Kubernetes 憑藉其快速增長和成熟的開源社區，成爲中國聯通的自然選擇。 中國聯通啓用 Kubernetes 的雲平臺現在託管着 50 個微服務，採用這種全新的開發模式向前推進。 “Kubernetes 改善了我們使用雲基礎設施的體驗，”張承宇說。“目前沒有其他技術可以替代它。” 中國聯通還使用 Istio 作爲其微服務框架， 另外還使用了 Envoy、CoreDNS 和 Fluentd。

影響

在中國聯通，Kubernetes 提高了運營和開發效率。資源利用率提高了 20-50%， 降低了 IT 基礎設施成本，部署時間從幾個小時縮短到 5-10 分鐘。 “這主要是因爲 Kubernetes 的自愈和可擴展性，促使我們提高了運維效率，”張承宇說。 “例如，我們目前只有五個人維護這麼多系統。我們從未想過能在這麼短的時間內實現這樣的可擴展性。”

張所在的團隊負責新技術、研發和平臺，開始尋找 IT 管理解決方案。中國聯通曾是一家完全國有的公司，近年來接受了來自 BAT（百度、阿里巴巴、騰訊）和京東的民營投資，現在專注於使用開源技術而不是商業產品進行本土開發。 因此，該團隊開始尋找適合其雲基礎設施的開源編排方案。

儘管中國聯通已經在覈心電信運營商系統中使用了 Mesos，但團隊認爲 Kubernetes 是新一代雲平臺的自然選擇。 “主要原因是 Kubernetes 有一個成熟的社區，”張承宇說。“它增長非常迅速，因此我們可以從其他人的最佳實踐中學習。” 中國聯通還使用 Istio 作爲其微服務框架，另外還使用了 Envoy、CoreDNS 和 Fluentd。

中國聯通啓用 Kubernetes 的雲平臺現在託管着 50 個微服務，採用這種全新的開發模式向前推進。 中國聯通的開發人員可以通過 API 輕鬆利用該技術，而無需自己進行開發。這個雲平臺提供了 20-30 個服務， 連接到公司的數據中心 PaaS 平臺，並支持爲中國 31 個省各地分公司的內部用戶提供大數據分析等功能。

“Kubernetes 改善了我們使用雲基礎設施的體驗，”張承宇說。“目前沒有其他技術可以替代它。”

事實上，Kubernetes 提升了中國聯通的運營和開發效率。資源利用率提高了 20-50%， 降低了 IT 基礎設施成本，部署時間從幾個小時縮短到 5-10 分鐘。 “這主要是由於 Kubernetes 的自愈和可擴展性，因此我們可以提高運維效率，”張承宇說。“例如，我們目前只有五個人就能維護這麼多系統。”

隨着中國聯通在 Kubernetes 上取得的成功，張承宇和他的團隊渴望回饋社區。 這從參與聚會和會議開始，併爲考慮類似路徑的其他公司提供建議。 “特別是對於那些擁有傳統雲計算系統的公司，我真得建議他們加入雲原生計算社區，”張承宇說。

{{< case-studies/quote author="賈傑，中國聯通平臺技術研發組成員" >}} “公司可以使用像 Rancher 這樣的公司提供的託管服務，因爲他們已經定製了這項技術，你可以輕鬆利用這項技術。” {{< /case-studies/quote >}}

平臺技術研發組成員賈傑補充道，儘管“這項技術相對複雜，但只要開發人員習慣它，他們就可以體會到所有的好處。” 張承宇根據自己使用虛擬機雲的經驗指出，“Kubernetes 和這些雲原生技術相對簡單。”

此外，“公司可以使用像 Rancher 這樣的公司提供的託管服務， 因爲他們已經定製了這項技術，”賈傑說。“你可以輕鬆利用這項技術。”

展望未來，中國聯通計劃在 Kubernetes 上開發更多應用，專注於大數據和機器學習。 該團隊正在繼續優化他們構建的雲平臺，並希望通過合規性測試以加入 CNCF 的認證 Kubernetes 合規性計劃（Certified Kubernetes Conformance Program）。 他們也希望有一天能夠將代碼回饋給社區。

聽起來是不是雄心勃勃，那是因爲他們採用 Kubernetes 後所獲得的成果超出了他們的最高預期。 張承宇說：“我們從未想過能在這麼短的時間內實現這樣的可擴展性。”
