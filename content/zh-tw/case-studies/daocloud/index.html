挑戰

作爲雲原生領域的創新領導者，DaoCloud 成立於 2014 年底， 擁有自主知識產權的核心技術，致力於打造開放的雲操作系統，爲企業數字化轉型賦能。

DaoCloud 從誕生之日起就在致力於佈道雲原生。由於容器化對於雲原生業務至關重要， 沒有容器作爲基礎設施的雲平臺不太可能吸引其潛在使用者。因此，DaoCloud 面臨的第一個挑戰就是如何高效地管理和調度衆多的容器，同時保持它們之間的穩定連接。

隨着雲原生技術的蓬勃發展，雲原生解決方案如雨後春筍般激增。然而，擁有更多選擇並不總是一件好事， 因爲從各種產品中進行選擇以在全球範圍內實現利益最大化和成本最小化總是具有挑戰性和要求的。 因此，擺在 DaoCloud 面前的另一個障礙是如何在各個領域中挑選出最好的跑者，並將他們組織到一個平臺上， 實現雲原生的全局最優。

解決方案

Kubernetes 作爲容器編排的事實標準，無疑是首選的容器解決方案。DaoCloud 架構師兼開源團隊負責人 Paco Xu 表示 “Kubernetes 是目前容器生態裏面比較基礎的一環，絕大多數服務都是基於 Kubernetes 部署的，應用絕大多數都是在 Kubernetes 叢集中運行和管理。“

面對層出不窮的技術方案，DaoCloud 研發副總裁 Peter Pan 認爲，“在衆多技術面前，堅持以 Kubernetes 爲核心，整合周邊最佳實踐和先進技術，打造一個適合的平臺和方案，纔是尋找全局最優解的正確路徑。”

影響

在擁抱雲原生的過程中，DaoCloud 不斷向 Kubernetes 等優秀的 CNCF 開源項目學習， 逐漸形成了以 DaoCloud Enterprise 雲原生應用雲平臺爲核心的產品架構。DaoCloud 堅持以 Kubernetes 等世界領先的雲原生技術爲支點，爲軍工、金融、製造、能源、政務、 零售等垂直行業提供了前沿的雲原生解決方案，爲浦發銀行、華泰證券、富國基金、上汽集團、 海爾、復旦大學、屈臣氏、吉致汽車金融、國家電網等各行各業的優秀企業都量身定製了滿意的數字化轉型方案。

{{< case-studies/quote image="/images/case-studies/daocloud/banner2.jpg" author="Kebe Liu， DaoCloud 服務網格專家" >}} “隨着 DaoCloud Enterprise 越來越強大，客戶覆蓋面越來越廣，有些客戶需要使用 Kubernetes 而不是 Swarm 進行應用編排。Kubernetes 作爲一套功能強大的應用編排系統，有着強大的社區支持， 也受到很多大公司的青睞。我們作爲提供商需要滿足使用者的需求。” {{< /case-studies/quote >}}

DaoCloud 成立的初衷就是幫助傳統企業進行數字化轉型，實現應用上雲。公司成立之後發佈的首秀產品 DaoCloud Enterprise 1.0 便是一款基於 Docker 的容器引擎平臺，可以輕鬆打包構建映像檔並運行容器。

但是，隨着應用數量的增加，容器越來越多，如何協調和調度這些容器逐漸成爲制約產品性能的主要瓶頸。 DaoCloud Enterprise 2.0 開始採用 Docker Swarm 管理容器，但隨着容器調度系統越來越複雜， Docker Swarm 也開始顯得力不從心。

此時恰逢 Kubernetes 嶄露頭角，憑藉多樣的功能、穩定的性能、及時的社區支持、 強大的兼容性等優勢迅速發展成爲容器編排的業界標準。Paco Xu 表示“企業容器平臺需要容器編排來規範化應用上雲的過程。 Kubernetes 在 2016 - 2017 年逐漸成爲容器編排的事實標準，我們在 2017 年就開始同時支持 Docker Swarm 和 Kubernetes 了。”

經過一系列的評估，2017 年發佈的 DaoCloud Enterprise 2.8 版本開始正式採用 Kubernetes（v1.6.7）作爲容器編排工具。此後，2018 年發佈的 DaoCloud Enterprise 3.0 採用 Kubernetes 1.10 版本，2021 年發佈的 DaoCloud Enterprise 4.0 採用 Kubernetes 1.18 版本。 2022 年發佈的 DaoCloud Enterprise 5.0 支持 Kubernetes 1.23 至 1.26 版本。

六年時間裏發佈的四個主要版本一直都在堅定不移地使用 Kubernetes，這足以說明當時的選擇是正確的。 DaoCloud 用實際經驗證明了 Kubernetes 是容器編排的最佳選擇，也用自身行動證明了自己一直都是 Kubernetes 的忠實擁躉。

{{< case-studies/quote image="/images/case-studies/daocloud/banner3.jpg" author="Ting Ye，DaoCloud 產品創新副總裁" >}} "Kubernetes 是將我們的產品改進爲世界一流軟體的基石。" {{< /case-studies/quote >}}

Kubernetes 幫助我們的產研團隊從 0 到 1 完成了從研發構建自動化，測試自動化，安全自動化， 發佈自動化保證了軟體交付質量，其次實現了智能化協作溝通，包括產品需求及定義體系、產品多語言協作體系、 產品缺陷修復協作體系、疑難雜症攻堅體系，極大的提升了產研同部門、跨部門的協作效率。

在 Kubernetes 的助力下，DaoCloud 的產品性能更優，更具競爭力。DaoCloud 堅持以 Kubernetes 爲核心，整合周邊最佳實踐和先進技術，打造出 DaoCloud Enterprise 雲原生應用雲平臺， 提供應用商店、應用交付、微服務治理、可觀測性、資料服務、多雲編排、信創異構、雲邊協同等能力。 DaoCloud Enterprise 5.0 是集雲原生技術大成的完全形態。

DaoCloud 爲上海浦發銀行部署 Kubernetes 平臺後，應用部署效率提升 82%，交付週期從半年縮短到一個月， 交易成功率達到 99.999%；

四川天府銀行落地基於 Kubernetes 的雲原生平臺，將彈性響應時間由數小時大幅縮減到平均 2 分鐘， 產品迭代週期從兩個月縮短爲兩週，應用上線時間縮短 76.76%。

爲某合資車企搭建基於 Kubernetes 的雲原生平臺後，將其交付週期從兩個月縮短到一兩週，應用部署成功率提升 53%，應用上線效率提高 24 倍；爲某跨國零售集團部署基於 Kubernetes 的多個雲原生平臺模塊，爲其減少了 46% 的應用部署問題，將監控定位效率提升 90% 以上。

爲某大型綜合類券商搭建統一的雲原生 PaaS 平臺，使其業務流程效率提升 30%，資源成本節約 35% 左右。

爲富國基金打造基於 Kubernetes 的新一代雲原生 PaaS 平臺，將標準中間件部署時間從數小時縮短至數分鐘， 中間件運維能力提升 50%，容器化程度提升 60%，資源利用率提升 40%。

另一方面，DaoCloud 自身的產品研發工作也是基於 Kubernetes 進行的。公司基於 Kubernetes 部署了 Gitlab，形成了 “Gitlab —> PR —> 自動化測試 —> 構建發佈“的產品開發流程，顯著提升了開發效率， 減少了重複測試的工作量，實現了應用的自動發佈。這樣一來，大大節省了運維成本，技術人員可以爲開發產品投入更多的時間與精力， 打磨出更優秀的雲原生產品。

{{< case-studies/quote image="/images/case-studies/daocloud/banner4.jpg" author="Paco Xu，DaoCloud 開源和高級開發團隊負責人" >}} “我們的開發者很踊躍地貢獻開源，沉澱技術實力，在 Kubernetes 和 Istio 社區都有越來越多的貢獻。 公司第五代產品走的也是開源路線，爲雲原生技術添磚加瓦，完善技術生態。” {{< /case-studies/quote >}}

DaoCloud 深度參與貢獻 Kubernetes 等多項雲原生開源項目，在雲原生開源社區中的參與度、貢獻度持續增長。 在過去一年裏，DaoCloud 在 Kubernetes 的開源榜單累計貢獻度位居全球第三（基於 Stackalytics 網站 2023/06/13 的資料）。

在 2022 年 8 月由 Kubernetes 官方組織的社區貢獻者訪談活動中，接見了來自亞太地區的 4 位優秀貢獻者， 其中 Shiming Zhang 和 Paco Xu 都來自 DaoCloud，二人均是 SIG Node 的 Reviewer。此外在 2022 Kubecon 北美站上，DaoCloud 的 Kante Yin 榮獲 Kubernetes 2022 年度貢獻者獎。

此外，DaoCloud 也在堅持踐行雲原生信仰，持續回饋雲原生社區，開源了 Clusterpedia、 Kubean、 CloudTTY、 KLTS、 Merbridge、 HwameiStor、 Spiderpool 和 KWOK 等優秀項目， 不斷完善 Kubernetes 生態體系。

其中：

  • Clusterpedia：Clusterpedia 兼容 Kubernetes OpenAPI， 實現了多叢集資源的同步，提供了更強大的搜索功能，可以快速、輕鬆、有效地獲取叢集內所有資源資訊。
  • Kubean：Kubean 支持快速創建 Kubernetes 叢集以及其他廠商的叢集。
  • CloudTTY：CloudTTY 是專爲 Kubernetes 雲原生環境打造的 Web 終端和 Cloud Shell Operator，可以通過一個 Web 頁面隨時隨地管理 Kubernetes 叢集。
  • KLTS：KLTS 爲 Kubernetes 早期版本提供長期免費的維護支持。
  • KWOK：Piraeus 是適用於 Kubernetes 的高性能、高可用性、簡單安全的儲存解決方案。

DaoCloud 融合自身在各行各業的實戰經驗，持續貢獻 Kubernetes 開源項目， 致力於讓以 Kubernetes 爲代表的雲原生技術更平穩、高效地落地到產品和生產實踐中。

{{< case-studies/quote image="/images/case-studies/daocloud/banner5.jpg" author="Song Zheng，DaoCloud 中國區技術總經理" >}} “DaoCloud 作爲首批 CNCF 官方認證的雲原生技術培訓夥伴，將持續開展賦能培訓、 項目指導等活動，攜手夥伴，爲客戶導入雲原生，共同打造雲原生能力的最佳實踐路徑。” {{< /case-studies/quote >}}

企業使用者需要的是一個全局最優解，這個最優解可以理解爲是涵蓋多雲編排、信創異構、 應用交付、可觀測性、雲邊協同、微服務治理、應用商店、資料服務等能力的最大公約數。 在如今的雲原生生態體系裏，這些功能都離不開 Kubernetes 作爲底層的容器編排技術。 這就意味着 DaoCloud 在尋找數字世界最優解的過程中也離不開 Kubernetes， 未來的產品研發也將繼續以 Kubernetes 爲基礎。

此外，DaoCloud 一直致力於 Kubernetes 的培訓、推廣活動。2017年，公司憑藉核心產品雲原生應用雲平臺 DaoCloud Enterprise 成爲全球首批通過 CNCF Kubernetes 兼容性認證的廠家。2018年，公司成爲 CNCF 認證的 Kubernetes 服務提供商，併成爲全球首批獲得CNCF官方認證的 Kubernetes培訓合作伙伴，全面擁抱 Kubernetes 技術生態。

2022 年 11 月 18 日，由 CNCF 和 DaoCloud 、華爲雲、四川天府銀行、 OPPO 聯合發起的「Kubernetes Community Days 成都站」成功舉辦， 聚集了來自雲原生領域開源社區的最終使用者、貢獻者和技術專家，分享關於雲原生的多行業實踐、 熱門開源項目、社區貢獻心得等豐富內容。未來，DaoCloud 將繼續爲 Kubernetes 貢獻自己的力量，通過項目培訓、社區貢獻等活動不斷擴大 Kubernetes 的影響力。
